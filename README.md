# IT Recruitment Optimization Framework (BPMN)

![screenshot](Proces_rekrutacji_IT_BPMN.png)

The **IT Recruitment Optimization Project** is designed to standardize and professionalize the collaboration between internal HR departments and external headhunting agencies. By implementing a structured **BPMN (Business Process Model and Notation)** framework, this project aims to eliminate common communication gaps, reduce "Time-to-Hire," and ensure a seamless experience for technical candidates.

The model specifically utilizes **BPMN gateways** to manage complex logic, such as feedback loops after technical tasks, candidate rejections, and multi-stage evaluation processes.

---

## Process Architecture

The framework utilizes a horizontal collaboration format, clearly defining responsibilities and hand-offs across four distinct lanes:

### 1. Hiring Manager (Technical Decision Maker)
* **Role:** Defines the technical requirements and "Must-Haves" for the role.
* **Key Tasks:** Technical task review, final interview participation, and the ultimate "Go/No-Go" decision.

### 2. Internal HR (The Bridge)
* **Role:** Orchestrates the entire process and manages the relationship with the agency.
* **Key Tasks:** Cultural fit assessment, contract negotiation, and ensuring the process stays within corporate compliance.

### 3. Headhunting Agency (Active Sourcing)
* **Role:** Acts as the external engine for candidate acquisition.
* **Key Tasks:** Active sourcing, initial technical vetting, and "pre-selling" the candidate on the company vision.

### 4. Candidate (The Journey)
* **Role:** The central subject of the process.
* **Key Tasks:** Progresses through the pipeline from first contact to technical task submission, final interviews, and offer acceptance.

---

## Key Features & BPMN Logic

* **Standardized Hand-offs:** Every transition between the Agency, HR, and the Hiring Manager is mapped to prevent candidates from "falling through the cracks."
* **Gateway Management:** Specialized logic for handling technical task evaluations—if a candidate fails a task, the process triggers an automated feedback loop rather than a simple dead-end.
* **Optimized Time-to-Hire:** By defining clear SLAs (Service Level Agreements) for feedback at each gateway, the model significantly accelerates the recruitment cycle.
* **Transparency:** Provides a visual roadmap of the applicant’s journey, making the process predictable for all parties involved.

---

## Expected Outcomes

| Metric | Objective |
| :--- | :--- |
| **Time-to-Hire** | Reduce the average duration from sourcing to offer by 20-30%. |
| **Candidate Experience** | 100% feedback rate for candidates post-technical evaluation. |
| **Agency Efficiency** | Higher "CV-to-Interview" ratio due to better initial technical alignment. |
| **Communication** | Elimination of redundant emails through structured BPMN-driven status updates. |
